# local-services

Process manager for local development services on macOS. Uses [Supervisor](http://supervisord.org/) under the hood ‚Äî a single launchd agent boots Supervisor at login and it manages everything defined in `conf.d/`. Comes with a menu bar app for quick status checks and start/stop/restart without opening a terminal.

## One-time setup

### 1. Install dependencies

```bash
brew install supervisor uv python@3.12
```

> **Why `python@3.12`?** The menu bar app uses macOS AppKit APIs which require a Homebrew "framework" Python. The Python bundled with `uv` does not work for GUI apps.

### 2. Run the install script

```bash
./install.sh
```

The script will:

1. Verify `supervisord`, `uv`, and a Homebrew framework Python are installed
2. Create log and socket directories (`~/Library/Logs/supervisor/`, `~/.supervisor/`)
3. Generate and load a launchd plist ‚Äî Supervisor starts immediately and auto-starts at every login
4. Add a `supervisorctl` alias to your `.zshrc` (or `.bash_profile`)
5. Loop over all `conf.d/*.conf.template` files, prompt you for each service's variables, and generate the corresponding `.conf` files
6. Generate `/Applications/LocalServices.app` ‚Äî a double-clickable app to launch the menu bar UI
7. Optionally register the menu bar app to launch at login

### 3. Reload your shell

```bash
source ~/.zshrc   # or ~/.bash_profile
```

### 4. Verify

```bash
supervisorctl status
```

You should see your services with status `RUNNING`.

## Menu bar app

The menu bar app shows the status of all services at a glance and lets you control them without a terminal.

### Launching it

**Option A ‚Äî Double-click** `/Applications/LocalServices.app` (or keep it in the Dock).

**Option B ‚Äî From terminal** (stays attached):

```bash
uv run --python /opt/homebrew/bin/python3 menubar/app.py
```

**Option C ‚Äî From terminal, detached** (survives closing the terminal window):

```bash
menubar/start.sh
```

### What the icon means

| Icon | Meaning |
|------|---------|
| `‚óè` | All services running |
| `‚óê` | Some services running |
| `‚óã` | All services stopped |
| `!` | supervisord is not reachable |

### What you can do from the menu

- **Click a service** ‚Üí submenu with Start / Stop / Restart
- **Reload Config** ‚Üí picks up new or changed `.conf` files (same as `supervisorctl reread && update`)
- **View Logs** ‚Üí opens a `tail -f` in Terminal for any service's stdout, stderr, or supervisord itself
- **Launch at Login** ‚Üí toggles a launchd plist so the menu bar app starts automatically at login (shows `‚úì` when active)
- **Quit** ‚Üí closes the menu bar app (services keep running)

## Day-to-day usage (terminal)

If you prefer the command line over the menu bar app:

```bash
supervisorctl status                        # check all services
supervisorctl start bookmark-processor      # start one
supervisorctl stop bookmark-processor       # stop one
supervisorctl restart bookmark-processor    # restart one
supervisorctl stop all                      # stop everything

# Live logs
tail -f ~/Library/Logs/supervisor/bookmark-processor.log
tail -f ~/Library/Logs/supervisor/bookmark-processor.error.log
```

## Adding a new service

1. Create `conf.d/my-app.conf.template` ‚Äî a standard Supervisor `[program:...]` config. Use `{{VARNAME}}` for any paths that differ per machine. Two placeholders are always available for free:
   - `{{UV_BIN}}` ‚Äî resolved to the path of `uv`
   - `{{REPO_DIR}}` ‚Äî resolved to the path of this repo

2. Create `conf.d/my-app.conf.vars` ‚Äî one line per variable:

   ```text
   VARNAME|Human-readable prompt|/default/path
   ```

3. Re-run `./install.sh` to generate the `.conf`, or fill in the values manually.

4. If Supervisor is already running, pick it up without restarting:

   ```bash
   supervisorctl reread && supervisorctl update
   ```

   Or click **Reload Config** in the menu bar app.

## Removing a service

1. Delete its `conf.d/<name>.conf` file (and optionally the `.conf.template` + `.conf.vars`)
2. Run `supervisorctl reread && supervisorctl update` or click **Reload Config** in the menu bar app
